How to Build a 90-Day Revenue Operating System

Every engagement I run follows the same three-phase structure: Diagnose, Design, Lead. Thirty days each, roughly. It's not a rigid formula - some companies need longer in Diagnose because the data doesn't exist yet, some skip ahead in Design because half the pieces are already there - but the sequence itself doesn't change, because you can't design a fix for a problem you haven't actually confirmed yet.

Days 1–30: Diagnose

The instinct is to start fixing immediately. Resist it. Most of what looks broken on day one turns out to be a symptom of something else by day twenty.

  • Pipeline audit. Every open deal gets scored against the buyer signal framework - timing, relevance, intent - not taken at face value.
  • Win/loss interviews. Talk to five recently closed-won and five closed-lost accounts. The pattern in why deals actually died is almost never what internal Slack threads assumed it was.
  • Message-market fit check. Does the pitch reps are actually using in live calls match what's converting, or has it drifted from what marketing thinks is being said?
  • Infrastructure check. Deliverability, CRM hygiene, and dashboard accuracy - the underlying systems that make everything above trustworthy or not.

Days 31–60: Design

Everything built here comes directly from what Diagnose surfaced - never from a generic template.

  • KPI dashboard, rebuilt around leading indicators - not the lagging ones everyone was already staring at.
  • The playbook, written down - extracted from whoever's actually converting best, not copied from a course.
  • A shared qualified-lead definition - built with sales and marketing in the same room, so it survives contact with reality.
  • Account prioritization framework - the buyer-signal scoring rubric, adapted to your specific sales motion.

Days 61–90: Lead

Design without adoption is a document nobody reads. This phase is entirely about making the new system the way work actually happens.

  • Train the team on the playbook - with live call coaching, not a slide deck read once in a meeting.
  • Run the weekly cadence - pipeline review against signal score, KPI review against leading indicators, every week, no exceptions for the first quarter.
  • Coach from data, not instinct - the whole point of Design was to make coaching conversations concrete instead of vibes-based.
  • Set the next 90 days - the operating system needs its own review cycle, or it decays back into whatever was happening before.

What day 90 should look like

A dashboard the team actually checks unprompted. A playbook new hires can ramp on without shadowing the founder for three months. A pipeline number people trust because it's built on scored signal, not hope. None of that requires more headcount - it requires the system existing at all.
